Metode Box-Jenkins: Analisis Deret Waktu ARIMA

Pendahuluan

Metode Box-Jenkins, yang dikembangkan oleh George Box dan Gwilym Jenkins, adalah sebuah pendekatan sistematis untuk membangun model ARIMA (Autoregressive Integrated Moving Average). Metodologi ini berfokus pada empat tahap berulang untuk mengidentifikasi model terbaik yang menggambarkan struktur data deret waktu dan menghasilkan peramalan yang akurat.

Tahap 1: Identifikasi Model Awal

Tahap ini adalah fondasi dari seluruh proses, di mana kita menggunakan alat visual untuk menentukan struktur model ARIMA (p, d, q) yang potensial. Tujuan utama adalah untuk memastikan data kita bersifat stasioner.

a. Plot Time Series

Membuat plot deret waktu adalah langkah pertama untuk melihat apakah data sudah stasioner atau belum, baik dari segi rata-rata maupun variansi. Data dikatakan stasioner jika rata-rata dan variansinya konstan seiring waktu.

- Jika data tidak stasioner dalam rata-rata, kita dapat membuatnya stasioner melalui proses differencing (d). Nilai 'd' adalah jumlah differencing yang diperlukan.

- Jika data tidak stasioner dalam variansi, kita dapat menggunakan transformasi Box-Cox untuk menstabilkannya. Jika data tidak stasioner di keduanya, stabilkan variansi terlebih dahulu, baru kemudian rata-ratanya.

b. Plot ACF (Autocorrelation Function)

Plot ACF membantu kita mengukur korelasi antara nilai deret waktu saat ini dengan nilai-nilai sebelumnya pada berbagai lag. Pola plot ACF digunakan untuk mengidentifikasi parameter Moving Average (MA), atau 'q'.

c. Plot PACF (Partial Autocorrelation Function)

Plot PACF mengukur korelasi antara nilai deret waktu saat ini dengan nilai-nilai sebelumnya, setelah efek dari lag di antaranya dihilangkan. Pola plot PACF digunakan untuk mengidentifikasi parameter Autoregressive (AR), atau 'p'.

Tahap 2: Penaksiran Parameter

Setelah mengidentifikasi model potensial, kita menaksir parameter model ARIMA tersebut. Metode yang umum digunakan adalah Maximum Likelihood Estimation (MLE).

Model umum ARIMA (p, d, q) dinyatakan sebagai:

$\phi(B) (1-B)^d Y_t = \theta(B) \epsilon_t$

dimana

$$\psi_p(B) = 1 - \psi_1 B - \psi_2 B^2 - \cdots - \psi_p B^p$$
$$\theta_q(B) = 1 - \theta_1 B - \theta_2 B^2 - \cdots - \theta_q B^q$$

Di mana \(B\) adalah operator lag: \(B^k Y_t = Y_{t-k}\).

Nilai parameter $\phi_1$ ini harus diestimasi dari data. Jika nilai $\phi_1$ mendekati 1, model tersebut memiliki unit root dan tidak stasioner, sehingga harus kembali ke tahap identifikasi.

Tahap 3: Pemeriksaan Diagnostik

Tahap ini berfungsi sebagai kontrol kualitas. Kita mengecek apakah model yang telah ditaksir sudah cukup baik untuk merepresentasikan data.

  • Uji Signifikansi Parameter: Menggunakan uji-t, kita periksa apakah taksiran parameter (misalnya, $\phi_1$) signifikan secara statistik (p-value < 0.05). Jika tidak, model perlu disesuaikan.
  • Uji Asumsi Residual: Kita perlu menguji bahwa sisaan (residual) model bersifat white noise (tidak memiliki pola, rata-rata nol, dan variansi konstan) dan berdistribusi normal.

    - Uji White Noise: Gunakan Uji Ljung-Box.

    - Uji Normalitas: Gunakan Uji Kolmogorov-Smirnov atau plot Q-Q.

Jika ada satu atau lebih asumsi yang tidak terpenuhi, kita harus kembali ke Tahap 1 untuk mencoba model ARIMA lain.

Tahap 4: Peramalan (Forecasting)

Jika semua uji diagnostik berhasil dilewati, model yang telah dibuat siap digunakan untuk peramalan nilai di masa depan. Peramalan ini juga akan menyertakan interval kepercayaan, yang memberikan rentang nilai di mana nilai aktual kemungkinan besar akan berada.
